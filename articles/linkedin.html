How I went from 24ms to 3ms in face recognition inference.

6 months ago I started porting a neural network from ONNX to C.

Week 1: naive port. 24ms. Works, but 7x slower than ONNX Runtime.

I profiled every operation. The surprise?

Matrix multiplication was only 6% of inference time.

The real killers:

→ Layer normalization — 17 calls, scalar loops
→ GELU activation — naive tanh through math.h
→ Memory transposes — layout conversions on every block

So I wrote custom SIMD kernels for each operation.
AVX2 for everything. AVX-512 where available.
INT8 quantized GEMM with per-channel scaling.

24ms → 8ms → 5ms → 3ms

The last 2ms took 4 months.
The first 16ms took 2 weeks.

That's optimization in a nutshell.

The hardest part of beating ONNX Runtime wasn't speed.
It was accuracy.

My C engine produced garbage embeddings.
Cosine similarity with the reference: 0.067
Should be: 1.0

I found 7 bugs through per-layer intermediate dumps, comparing every tensor against a NumPy reference.

The most subtle:

① XCA attention computed full C×C matrices instead of per-head [head_dim × head_dim]. The reshape was hidden in the ONNX graph.

② Position embedding was treated as a function of input. It's actually a CONSTANT — computed once at load.

③ Depthwise conv was independent per-channel. ONNX cascades: conv0 → result → conv1.

Each fix: +0.1 cosine similarity.
Seven fixes later: 1.0

Lesson: never trust your understanding of a computational graph. Dump intermediates. Compare everything.
